## 一、知识点讲解：从零理解微信小程序

这一部分按“先讲小程序，再用 Vue/HTML 帮你类比”的方式整理。你不需要先会小程序，只要把它理解成一个运行在微信里的前端应用：页面结构、样式、数据、事件、组件、生命周期这些思想和 Vue 很像，但具体写法和运行环境换成了微信小程序自己的规则。

### 1. 小程序到底是什么

微信小程序是运行在微信里的轻量应用。用户不需要像安装 App 那样下载安装，可以通过扫码、搜索、分享、公众号关联等入口打开。它适合做轻量服务，比如点餐、查课表、预约、简单商城、班级通知等。

如果用 Vue 类比，小程序有点像一个被微信托管的前端项目。Vue 项目运行在浏览器中，小程序运行在微信这个“宿主环境”中。宿主环境可以理解为微信给小程序准备的一套运行空间，它提供了扫码、获取位置、收货地址、拨打电话、支付、分享等能力。

但小程序不能完全替代原生 App。原因是它受微信平台规则、权限、性能和系统能力限制。比如一个需要大量底层系统权限、极致性能或复杂后台常驻能力的应用，通常还是更适合原生 App。

考试要记：

| 考点             | 正确理解                                       |
| ---------------- | ---------------------------------------------- |
| 小程序特点       | 无需下载安装、使用方便、入口多、依托微信运行。 |
| 是否跨平台       | 通常支持 Android、iOS 等微信环境。             |
| 是否完全替代 App | 不能完全替代。                                 |
| 是否等同 WebApp  | 不等同，运行环境、入口和能力不同。             |
| 是否需要审核发布 | 需要上传代码、提交审核、审核通过后发布。       |

### 2. 小程序的运行结构：渲染层和逻辑层

小程序不是把所有代码都放在一个浏览器页面里运行。它大致分成两部分：

| 层     | 负责什么                 | 文件/内容                     |
| ------ | ------------------------ | ----------------------------- |
| 渲染层 | 显示界面、呈现结构和样式 | WXML、WXSS、组件              |
| 逻辑层 | 处理数据、事件、API 调用 | JS、`App()`、`Page()`、`wx.*` |

这点对理解 `setData()` 很重要。小程序的页面显示在渲染层，JS 逻辑运行在逻辑层，两边通信需要微信客户端做中转。所以逻辑层的数据变化不能只靠随手改变量，还要通过 `setData()` 把变化同步给视图。

Vue 类比：Vue 里你修改 `ref` 或响应式对象，框架会自动帮你更新模板；小程序里考试重点是 `this.setData({ key: value })`，它像是在明确告诉页面：“这个数据变了，去刷新对应视图。”

### 3. 开发流程和开发者工具

开发微信小程序一般要用微信开发者工具。它类似 Vue 项目中的 VS Code + 浏览器开发者工具 + 本地预览环境的组合。

常见流程：

| 步骤                | 说明                               |
| ------------------- | ---------------------------------- |
| 注册账号/获取 AppID | 小程序账号和公众号账号不是一回事。 |
| 创建项目            | 在微信开发者工具中创建小程序项目。 |
| 编写代码            | 写页面结构、样式、逻辑和配置。     |
| 本地预览调试        | 在模拟器和手机端预览。             |
| 上传代码            | 通过开发者工具上传体验版。         |
| 提交审核            | 到微信公众平台提交审核。           |
| 发布上线            | 审核通过后发布。                   |

开发者工具面板也常考：

| 面板    | 类比 Vue/浏览器开发经验 | 作用                 |
| ------- | ----------------------- | -------------------- |
| 模拟器  | 浏览器页面预览          | 看手机端效果。       |
| 编辑器  | VS Code 编辑区          | 写代码。             |
| Console | 浏览器 Console          | 看日志和错误。       |
| Source  | 浏览器 Sources          | 调试源码。           |
| AppData | Vue Devtools 的状态查看 | 看页面 data 数据。   |
| Network | 浏览器 Network          | 看请求、响应、耗时。 |

体验成员只适合体验版本，通常没有后台管理、上传代码、成员管理等权限。题库里问体验成员权限时，优先记“仅可使用体验版，无项目后台操作权限”。

### 4. 项目文件结构：小程序没有 `.vue`，而是页面四件套

Vue 里常见一个 `.vue` 单文件组件：

```vue
<template>页面结构</template>
<script>页面逻辑</script>
<style>页面样式</style>
```

小程序把这些拆成多个文件：

| 小程序文件   | 作用                       | Vue/HTML 类比                  |
| ------------ | -------------------------- | ------------------------------ |
| 页面 `.wxml` | 写页面结构                 | Vue 的 `<template>`，HTML 结构 |
| 页面 `.wxss` | 写页面样式                 | Vue 的 `<style>`，CSS          |
| 页面 `.js`   | 写页面数据、事件、生命周期 | Vue 的 `<script>`              |
| 页面 `.json` | 写当前页面配置             | Vue 中没有完全等价物           |

整个小程序还有全局文件：

| 文件/目录             | 作用                                    | 必背点                     |
| --------------------- | --------------------------------------- | -------------------------- |
| `app.js`              | 小程序入口逻辑，注册 `App()`            | 题库口径中属于必须文件。   |
| `app.json`            | 全局配置，配置页面路径、窗口、tabBar 等 | 题库口径中属于必须文件。   |
| `app.wxss`            | 全局样式                                | 不是题库口径中的必须文件。 |
| `project.config.json` | 开发者工具项目配置                      | 用于保存项目设置。         |
| `pages/`              | 存放页面文件                            | 每个页面常有四件套。       |

`app.json` 的 `pages` 数组很重要。它列出小程序有哪些页面，第一项通常就是首页。比如：

```json
{
  "pages": [
    "pages/index/index",
    "pages/detail/detail"
  ]
}
```

这可以类比 Vue Router 的路由表，但不要完全等同。Vue Router 用 `routes` 管页面路由，小程序用 `app.json` 配页面路径，再用 `wx.navigateTo` 等 API 跳转。

### 5. WXML：小程序的页面结构

WXML 是小程序的结构语言。你可以把它理解为“小程序版 HTML”或“小程序版 Vue template”。它负责告诉页面要显示哪些组件、文本、图片、列表和条件内容。

例子：

```xml
<view class="card">
  <text>{{title}}</text>
  <button bindtap="handleTap">点击</button>
</view>
```

这里的 `view` 类似 HTML 里的 `div`，`text` 类似文本标签，`button` 类似按钮。`{{title}}` 和 Vue 里的 `{{ title }}` 很像，都是把数据展示到页面上。

WXML 常考语法：

| 功能     | 小程序写法           | Vue 类比               | 说明                           |
| -------- | -------------------- | ---------------------- | ------------------------------ |
| 数据绑定 | `{{message}}`        | `{{ message }}`        | 把 data 中的数据展示出来。     |
| 条件渲染 | `wx:if="{{show}}"`   | `v-if="show"`          | 条件不满足时节点不渲染或销毁。 |
| 显示隐藏 | `hidden="{{!show}}"` | `v-show="show"`        | 节点还在，只是隐藏。           |
| 列表渲染 | `wx:for="{{list}}"`  | `v-for="item in list"` | 循环渲染列表。                 |
| 点击事件 | `bindtap="fn"`       | `@click="fn"`          | 点击后执行 JS 函数。           |
| 阻止冒泡 | `catchtap="fn"`      | `@click.stop="fn"`     | 执行事件并阻止继续冒泡。       |

`wx:if` 和 `hidden` 是高频题。`wx:if` 类似 Vue 的 `v-if`，会控制节点是否创建；`hidden` 类似 Vue 的 `v-show`，节点仍存在，只是显示或隐藏。因此，不频繁切换的内容适合 `wx:if`，频繁切换的内容更适合 `hidden`。

`block` 也很容易考。`block` 只是一个包装标签，不是组件，不会渲染成真实节点。可以把它想成 Vue 里的 `<template>` 包裹：只是帮你组织结构，不是真的显示一个标签。

### 6. WXSS：小程序的样式

WXSS 是小程序的样式语言，可以理解为“小程序版 CSS”。它兼容 CSS 的大部分写法，也有自己的扩展。

常考点：

| 考点         | 解释           | Vue/CSS 类比               |
| ------------ | -------------- | -------------------------- |
| `app.wxss`   | 全局样式       | 全局 CSS                   |
| 页面 `.wxss` | 当前页面样式   | Vue 组件中的局部样式       |
| `@import`    | 导入公共样式   | CSS 的 `@import`           |
| `rpx`        | 响应式尺寸单位 | 类似为移动端适配准备的单位 |
| Flex         | 弹性布局       | 和 CSS Flex 思想一致       |

题库常考“WXSS 只支持 rpx 一种单位”是错的。WXSS 不只支持 `rpx`，也可以使用 `px` 等单位。

`rpx` 主要用于适配不同屏幕。题库里常见 iPhone6 口径：`1rpx = 0.5px`。这不是让你做复杂换算，而是提醒你 `rpx` 是响应式单位。

样式优先级也要记：

```text
内联样式 > 页面 wxss > 全局 app.wxss
```

Flex 布局的判断题也很常见：只有父元素设置了 `display:flex`，相关 flex 属性才有意义。

### 7. JS、Page 和数据更新

小程序页面的 JS 文件通常用 `Page()` 注册页面。它负责页面数据、事件函数和生命周期。

```javascript
Page({
  data: {
    message: 'hello'
  },
  changeText() {
    this.setData({
      message: 'changed'
    })
  }
})
```

可以这样和 Vue 对照：

| Vue                                | 小程序               | 说明                            |
| ---------------------------------- | -------------------- | ------------------------------- |
| `data()` 或 `ref/reactive`         | `Page({ data: {} })` | 定义页面初始数据。              |
| `methods` 中的方法                 | `Page({ fn() {} })`  | 定义事件处理函数。              |
| 修改响应式数据                     | `this.setData()`     | 小程序通过 `setData` 同步视图。 |
| `<template>{{message}}</template>` | WXML `{{message}}`   | 页面展示数据。                  |

注意：考试里不要写“直接改 `this.data.message` 就能刷新页面”。小程序中逻辑层数据同步到视图层，重点记 `setData()`。

### 8. 生命周期：什么时候自动执行哪些函数

生命周期就是“小程序运行到某个阶段时自动调用的函数”。Vue 有 `mounted`、`unmounted` 等，小程序也有类似思想，但分成应用生命周期和页面生命周期。

应用生命周期写在 `App()` 中，关注整个小程序：

| 函数       | 什么时候执行             | 类比理解                                 |
| ---------- | ------------------------ | ---------------------------------------- |
| `onLaunch` | 小程序初始化时执行       | 整个应用第一次启动，适合初始化全局数据。 |
| `onShow`   | 小程序启动或从后台回前台 | 应用显示。                               |
| `onHide`   | 小程序从前台进入后台     | 应用隐藏。                               |

页面生命周期写在 `Page()` 中，关注某一个页面：

| 函数       | 什么时候执行     | 常见用途                       |
| ---------- | ---------------- | ------------------------------ |
| `onLoad`   | 页面加载时       | 获取页面参数、初始化页面数据。 |
| `onShow`   | 页面显示时       | 每次回到页面可刷新数据。       |
| `onReady`  | 页面初次渲染完成 | 做渲染完成后的操作。           |
| `onHide`   | 页面隐藏时       | 暂停操作、保存状态。           |
| `onUnload` | 页面卸载时       | 清理资源。                     |

Vue 类比时可以粗略记：`onLoad` 像页面创建时初始化，`onReady` 像首次渲染完成，`onUnload` 像页面销毁。但考试一定写小程序自己的函数名。

高频判断：`onLaunch` 是 App 生命周期函数，不是 Page 生命周期函数；`onLaunch` 不会在每次页面加载时重复执行。

### 9. 事件：用户点了、滑了、输入了怎么办

事件就是用户操作后触发的函数。Vue 中常写 `@click="handleClick"`，小程序里常写 `bindtap="handleTap"`。

```xml
<button bindtap="add">加一</button>
```

```javascript
Page({
  data: {
    count: 0
  },
  add() {
    this.setData({
      count: this.data.count + 1
    })
  }
})
```

常考事件：

| 小程序事件/写法 | 作用                   | Vue 类比      |
| --------------- | ---------------------- | ------------- |
| `tap`           | 点击/单击事件          | `click`       |
| `bindtap`       | 绑定点击事件，会冒泡   | `@click`      |
| `catchtap`      | 绑定点击事件，阻止冒泡 | `@click.stop` |
| `touchmove`     | 手指移动               | 触摸移动事件  |
| `touchend`      | 触摸结束               | 触摸结束事件  |

题库里“使用 bind 绑定事件，会触发事件冒泡”是对的。

### 10. 组件：页面里能摆放的积木

组件就是页面上的基础积木。Vue 里你会写 HTML 标签和自定义组件；小程序里大量使用微信提供的内置组件。

| 小程序组件    | 它是什么      | HTML/Vue 类比 | 常考点                          |
| ------------- | ------------- | ------------- | ------------------------------- |
| `view`        | 基础布局容器  | `div`         | 小程序界面布局基础。            |
| `text`        | 文本组件      | `span/p`      | `selectable` 控制文本是否可选。 |
| `input`       | 单行输入框    | `input`       | 可设置 `placeholder`、禁用等。  |
| `form`        | 表单          | `form`        | 表单控件通常要设置 `name`。     |
| `button`      | 按钮          | `button`      | 可用于提交、重置、触发事件。    |
| `scroll-view` | 可滚动区域    | 可滚动容器    | `scroll-y` 开启纵向滚动。       |
| `swiper`      | 轮播/滑块视图 | 轮播组件      | 常用于 banner、商品图。         |
| `navigator`   | 页面跳转组件  | `router-link` | `open-type` 控制跳转方式。      |
| `image`       | 图片组件      | `img`         | 常考缩放模式。                  |
| `video`       | 视频组件      | `video`       | 播放视频，不是音频。            |
| `audio`       | 音频组件      | `audio`       | 播放音频，不是视频。            |
| `progress`    | 进度条        | progress      | 默认长度单位不是 rpx。          |

所有组件和属性名称通常使用小写。题库中“所有组件都使用大写”这种说法是错的。

`image` 缩放模式常见有 `scaleToFill`、`widthFix`、`aspectFit`。`scroll-view` 中 `bindscrolltolower` 表示滚动到底部触发，可以类比列表触底加载更多。

### 11. WXS：运行在视图层的小脚本

WXS 可以理解为小程序视图层里的轻量脚本，常用于格式化数据、过滤列表、排序等。它和页面 JS 不是一个运行环境。

Vue 类比时，不要把 WXS 完全当成 `methods` 或 `computed`。Vue 的 `computed/methods` 可以访问组件逻辑；小程序的 WXS 与逻辑层 JS 隔离，不能直接调用 `wx` API，也不能直接调用 `Page` 中定义的函数。

考试记法：

| 考点         | 正确理解                                 |
| ------------ | ---------------------------------------- |
| WXS 运行位置 | 视图层。                                 |
| 与 JS 关系   | 与逻辑层 JS 隔离。                       |
| 典型用途     | 格式化、过滤、排序等展示辅助逻辑。       |
| 模块化       | 可通过 `module.exports` 暴露变量和函数。 |
| 不能做什么   | 不能直接调用 `wx` API 或 Page 函数。     |

### 12. 模板引用：import 和 include

小程序里可以复用 WXML 片段。这里最容易混的是 `import` 和 `include`。

| 写法      | 作用                                        | 类比理解                   |
| --------- | ------------------------------------------- | -------------------------- |
| `import`  | 主要引用目标文件中定义的 `template` 模板    | 像引入一个可复用模板。     |
| `include` | 引入目标文件除 `<template/>` 之外的整体代码 | 像把一段页面片段复制进来。 |

题库判断：“`import` 引用模板文件，`include` 可以将目标文件除 `<template/>` 之外的整个代码引入”是对的。

另外要避免循环引用，比如 A 引 B，B 又引 A。`import` 也不要理解为可以无限递归使用被引入文件中再引入的模板。

### 13. 常用 API：小程序里的 `wx.*`

Vue 项目里你可能用 `axios/fetch/localStorage/router`。小程序里很多能力由微信统一封装成 `wx.*` API。

网络请求：

| 小程序       | Vue/Web 类比      | 说明               |
| ------------ | ----------------- | ------------------ |
| `wx.request` | `axios` / `fetch` | 发起网络请求。     |
| `success`    | `then`            | 成功回调。         |
| `fail`       | `catch`           | 失败回调。         |
| `complete`   | `finally`         | 成功失败都会执行。 |

请求相关常考点：

| 考点         | 正确理解                                        |
| ------------ | ----------------------------------------------- |
| `wx.request` | 发起 HTTPS 网络请求。                           |
| `url`        | 可以带端口号。                                  |
| `header`     | 默认常见值为 `content-type: application/json`。 |
| `Referer`    | 小程序请求头中不能自行设置。                    |
| `complete`   | 不论成功或失败都会执行。                        |

缓存相关：

| 小程序 API              | Vue/Web 类比           | 说明                                    |
| ----------------------- | ---------------------- | --------------------------------------- |
| `wx.setStorage`         | `localStorage.setItem` | 异步保存缓存。                          |
| `wx.getStorage`         | `localStorage.getItem` | 异步读取缓存。                          |
| `wx.getStorageInfoSync` | 同步读取缓存信息       | 同步 API，常用 `try...catch` 捕获异常。 |

位置、地址和登录：

| API/字段           | 作用                    | 易错点                                 |
| ------------------ | ----------------------- | -------------------------------------- |
| `wx.login`         | 获取临时登录凭证 `code` | 不是直接拿 `openid` 或 `session_key`。 |
| `wx.getLocation`   | 获取用户地理位置        | 需要关注 `scope.userLocation` 授权。   |
| `wx.chooseAddress` | 选择用户收货地址        | 常用于外卖、电商地址填写。             |

交互、媒体、设备：

| API                   | 作用               | 类比             |
| --------------------- | ------------------ | ---------------- |
| `wx.showToast`        | 轻提示             | UI 库 toast      |
| `wx.showModal`        | 模态对话框         | confirm 弹窗     |
| `wx.showActionSheet`  | 操作菜单           | action sheet     |
| `wx.chooseImage`      | 选择图片           | 文件选择         |
| `wx.uploadFile`       | 上传文件           | 表单文件上传     |
| `wx.previewImage`     | 图片预览           | 图片预览组件     |
| `wx.getImageInfo`     | 获取图片信息       | 读取图片元数据   |
| `wx.openLocation`     | 打开位置           | 地图定位         |
| `wx.scanCode`         | 扫码               | 设备能力         |
| `wx.makePhoneCall`    | 拨打电话           | 设备能力         |
| `wx.getSavedFileList` | 获取已保存文件列表 | 不是所有临时文件 |

### 14. 页面跳转：不是 Vue Router，但可以类比

Vue 中页面跳转常用 Vue Router，比如 `router.push()`、`router.replace()`。小程序没有 Vue Router，而是用页面栈和导航 API。

| 小程序 API      | 作用                             | Vue Router 类比  | 高频易错                      |
| --------------- | -------------------------------- | ---------------- | ----------------------------- |
| `wx.navigateTo` | 保留当前页，跳转到非 tabBar 页面 | `router.push`    | 不能跳 tabBar 页面。          |
| `wx.redirectTo` | 关闭当前页，跳转到某页面         | `router.replace` | 当前页面会被关闭。            |
| `wx.switchTab`  | 跳转到 tabBar 页面               | 切换底部导航页   | 路径后不能带普通 query 参数。 |
| `wx.reLaunch`   | 关闭所有页面，打开某页面         | 重置路由栈       | 页面栈被清空。                |

看到题目说“跳转到 tabBar 页面”，优先想到 `wx.switchTab`；看到“保留当前页面跳转普通页面”，想到 `wx.navigateTo`。

`navigator` 组件也可以做页面跳转，它的 `open-type` 常见值有 `navigate`、`redirect`、`switchTab`。

### 15. 分享与页面事件

小程序天然支持微信分享。页面中可以通过 `onShareAppMessage` 自定义转发标题、图片和路径。

Vue 类比：这不像普通 Vue 页面中随便写一个按钮就能完成微信分享，它是小程序宿主环境提供的能力，要按小程序的页面事件规则来写。

考试记法：

| 事件                | 作用                             |
| ------------------- | -------------------------------- |
| `onShareAppMessage` | 自定义分享标题、图片、页面路径。 |
| `onPullDownRefresh` | 页面下拉刷新事件。               |

题库中“`onShareAppMessage` 分享事件可自定义转发标题、图片、页面路径”是对的。

### 16. 云开发：不用自己搭后端的方案

云开发可以理解为微信给小程序提供的一套后端能力，常见包括云函数、云数据库、云存储等。它适合轻量项目快速上线，特别是没有备案域名、没有自建服务器、又想尽快做出功能的时候。

Vue 类比：如果你写 Vue 项目，通常还要自己准备后端接口、数据库、服务器。小程序云开发相当于平台帮你提供一部分后端能力，让你少搭很多基础设施。

考试判断：

| 场景                                     | 是否适合云开发       |
| ---------------------------------------- | -------------------- |
| 无备案域名、无服务器、轻量小程序快速上线 | 适合。               |
| 需要深度系统权限的应用                   | 不适合作为唯一理由。 |
| 日活超千万的大型复杂电商全量系统         | 不适合简单套用。     |
| 对性能要求极高的大型竞技小游戏           | 不适合简单套用。     |

### 17. 主观题答题思路

主观题不要只写“像 Vue”，一定要回到小程序术语。可以按这个模板答：

| 题目类型     | 答题框架                                                     |
| ------------ | ------------------------------------------------------------ |
| 解释某概念   | 先说它是什么，再说在哪个文件/哪个层使用，最后说作用。        |
| 比较两个概念 | 写共同点，再写区别，再写适用场景。                           |
| 分析功能实现 | 写页面结构、数据、事件、API、生命周期、异常处理。            |
| 和 Vue 类比  | 类比只用于辅助理解，答案主体必须写 WXML、WXSS、Page、App、`wx.*` 等小程序术语。 |

例如问“解释数据绑定”：可以答“小程序通过 WXML 的 `{{}}` 把 `Page` 中 `data` 的数据展示到页面，用户事件触发 JS 方法后，通过 `setData()` 修改数据并同步到视图。它和 Vue 的模板插值很像，但小程序考试重点是 `setData()`。”

## 第一部分：复习要点

### 一、小程序基础与框架

#### 1. 小程序与传统 APP 的对比优势

- 无需安装卸载，即用即走。
- 占用存储空间小。
- 依赖微信生态，入口丰富，便于搜索、扫码、分享和传播。
- 可以调用微信能力，如登录、支付、定位、扫码、相机、分享等。
- 注意：小程序仍然依赖网络环境，并不是完全离线应用。

#### 2. 项目结构与配置文件

- `project.config.json`：项目配置文件，包括 `AppID`、项目名称、开发者工具设置等。
- `app.json`：全局配置文件，包括页面路径 `pages`、窗口样式 `window`、底部导航 `tabBar`、网络超时等。
- `app.js`：全局逻辑文件，用 `App()` 注册小程序实例，常见生命周期有 `onLaunch`、`onShow`、`onHide`。
- `app.wxss`：全局样式文件，页面自己的 `.wxss` 优先级更高，可覆盖全局样式。

#### 3. 页面组成

每个页面通常由四类文件组成：

- `.wxml`：页面结构，类似 Vue/HTML 中的模板部分。
- `.wxss`：页面样式，类似 CSS。
- `.js`：页面逻辑，定义 `Page()`、`data`、生命周期和事件处理函数。
- `.json`：页面配置，可选，用于覆盖 `app.json` 中部分窗口配置。

常见类比：

- `view` 类似 HTML 的 `div`。
- `text` 类似文本标签。
- `WXML` 类似 Vue 的模板。
- `WXSS` 类似 CSS。
- `data` + `{{ }}` 类似 Vue 的数据绑定。

#### 4. 生命周期

App 生命周期：

- `onLaunch`：小程序首次启动时触发，只触发一次。
- `onShow`：小程序从后台进入前台时触发。
- `onHide`：小程序从前台进入后台时触发。

Page 生命周期：

- `onLoad`：页面加载时触发，可获取路由参数。
- `onShow`：页面显示时触发，每次显示都会触发。
- `onReady`：页面初次渲染完成时触发，只触发一次。
- `onHide`：页面隐藏时触发，例如跳转到其他页面。
- `onUnload`：页面卸载时触发，例如页面被关闭或重定向。
- `onPullDownRefresh`：用户下拉刷新页面时触发。

第一次打开小程序首页的常见顺序：

`App.onLaunch` -> `App.onShow` -> `Page.onLoad` -> `Page.onShow` -> `Page.onReady`

#### 5. 页面栈与路由

- 小程序页面栈最多 10 层，超出后无法继续使用 `wx.navigateTo` 打开新页面。
- `wx.navigateTo`：保留当前页面，跳转到非 `tabBar` 页面，可返回。
- `wx.redirectTo`：关闭当前页面，跳转到非 `tabBar` 页面，不能返回原页面。
- `wx.reLaunch`：关闭所有页面，打开应用内某个页面。
- `wx.switchTab`：跳转到 `tabBar` 页面，并关闭其他非 `tabBar` 页面。
- `wx.navigateBack`：返回上一页或指定层级，常配合 `delta` 使用。

易错点：

- `wx.navigateTo` 不能跳转到 `tabBar` 页面。
- 跳转 `tabBar` 页面应使用 `wx.switchTab`。
- `wx.switchTab` 的路径后不能带查询参数。

#### 6. 数据绑定与事件

- 数据绑定使用 `{{ variable }}`。
- 数据绑定支持表达式和三元运算，但不能直接调用普通 JS 函数，复杂处理可借助 WXS。
- `bind事件名`：绑定事件，允许事件冒泡。
- `catch事件名`：绑定事件，并阻止事件冒泡。
- 常用事件有 `tap`、`input`、`change`、`submit`、`scroll`。
- 逻辑层修改页面数据必须使用 `this.setData()`，否则视图层不会自动更新。

#### 7. 条件渲染与循环渲染

- `wx:if`：条件为假时不渲染，适合切换频率低的场景。
- `hidden`：组件始终渲染，只是隐藏，适合频繁切换显示状态。
- `wx:for`：循环渲染数组。
- `wx:key`：循环渲染时建议指定，用于提升渲染性能。
- `wx:for-item`：指定当前数组元素变量名。
- `wx:for-index`：指定当前索引变量名。
- `<block>`：包装元素，不会渲染成实际节点，常用于包裹 `wx:if` 或 `wx:for`。

类比记忆：

- `<block>` 很像 Vue 中的 `<template>`，只起包裹作用，不产生真实页面节点。

#### 8. WXS

- WXS 全称 WeiXin Script。
- WXS 运行在视图层，不依赖逻辑层通信。
- 典型场景包括时间格式化、金额转换、字符串处理、列表数据过滤等。
- 使用时需在 WXML 中引入 WXS 模块，例如 `{{ wxs.formatTime(timestamp) }}`。

#### 9. 模板 template

- 定义模板：`<template name="card"> ... </template>`。
- 引用模板：`<import src="..." />` 加 `<template is="card" data="{{...}}" />`。
- 模板适合静态片段复用。
- 模板不具备独立的数据监听和生命周期。

#### 10. 全局数据与本地存储

- `getApp().globalData`：用于访问 `app.js` 中定义的全局共享数据。
- `globalData` 不会持久化，小程序关闭后可能丢失。
- 本地存储 API 包括 `wx.setStorageSync`、`wx.getStorageSync`、`wx.removeStorageSync`、`wx.clearStorageSync`。
- 异步本地存储 API 包括 `wx.setStorage`、`wx.getStorage`、`wx.removeStorage`、`wx.clearStorage`。
- 本地存储容量有限，常见总上限约 10MB。
- 清除所有本地缓存使用 `wx.clearStorage` 或 `wx.clearStorageSync`。

#### 11. 常用配置与限制

- `rpx` 适配以 iPhone6 屏幕宽度为基准，满屏宽度为 `750rpx`。
- `tabBar.list` 最少 2 项，最多 5 项。
- 小程序发布前通常需要提交审核，通过后才能上线。
- 小程序版本常见有开发版、体验版、正式版。
- 体验版用于内部测试，可添加体验成员，但体验成员数量受平台限制。

### 二、小程序组件

#### 1. 基础组件

- `view`：容器组件，类似 HTML 的 `div`。
- `text`：文本组件，支持文本展示；如需长按复制，要关注组件属性和基础库支持情况。
- `button`：按钮组件，支持 `open-type`，可用于分享、客服、获取手机号等能力。
- `input`：输入框组件，`type` 可控制键盘类型，如 `text`、`number`、`digit`、`idcard`。
- `image`：图片组件，支持 `mode` 缩放裁剪模式。
- `swiper`：轮播图容器，通常配合 `swiper-item` 使用。
- `scroll-view`：可滚动视图，垂直滚动时通常需要有确定高度。

#### 2. 媒体与地图组件

- `map`：地图组件，需要设置经纬度、标记点等。
- `canvas`：画布组件。
- `video`：视频播放组件。

#### 3. 表单与选择器组件

- `picker`：滚动选择器，`mode` 包括 `selector`、`multiSelector`、`time`、`date`、`region`。
- `checkbox`：多选组件。
- `radio`：单选组件。
- `form`：表单容器，常配合 `button form-type="submit"` 或 `button form-type="reset"`。
- `switch`：开关选择器。

### 三、小程序 API

#### 1. 网络请求

- `wx.request`：发起普通 HTTPS 网络请求。
- 正式环境需要在小程序管理后台配置合法请求域名。
- 常见回调包括 `success`、`fail`、`complete`。
- `complete` 无论成功还是失败都会执行。
- 官方网络能力默认超时时间为 60 秒，可在 `app.json` 或接口调用时配置。
- `wx.request`、`wx.uploadFile`、`wx.downloadFile` 的最大并发限制为 10 个。
- 小程序进入后台超过 5 秒且请求未结束时，可能触发 `fail interrupted`。

`wx.request` 示例：

```js
wx.request({
  url: 'https://www.test.com',
  method: 'GET',
  data: {
    school: '浙江水利水电学院'
  },
  success(res) {
    console.log(res.data)
  },
  fail(err) {
    console.log(err)
  },
  complete() {
    console.log('request complete')
  }
})
```

#### 2. 文件上传下载

- `wx.uploadFile`：上传本地资源到服务器，常用于上传图片、视频、文档等。
- `wx.downloadFile`：下载文件资源，成功后返回临时文件路径 `tempFilePath`。
- 上传下载都属于网络请求能力，需要注意合法域名、默认超时和并发限制。
- 下载得到的是临时文件路径，如需长期保存，应继续使用 `wx.saveFile` 或文件系统能力。
- 官方文档中 `wx.downloadFile` 单个下载文件也有大小限制，复习时重点先记“返回临时文件路径 + 可配合打开/保存”。

上传示例：

```js
wx.uploadFile({
  url: 'https://example.com/upload',
  filePath: tempFilePaths[0],
  name: 'file',
  formData: {
    extra: 'value'
  },
  success(res) {
    console.log(res.data)
  }
})
```

下载示例：

```js
wx.downloadFile({
  url: 'https://example.com/file',
  success(res) {
    const filePath = res.tempFilePath
    console.log(filePath)
  }
})
```

考试记忆点：

- `uploadFile` 关键参数：`url`、`filePath`、`name`、`formData`。
- `downloadFile` 成功回调中常用 `res.tempFilePath` 获取临时文件路径。
- 下载、上传和普通请求最大并发均为 10 个，默认超时 60 秒。

#### 3. WebSocket 会话

- `wx.connectSocket`：创建 WebSocket 连接，地址必须使用 `wss://`。
- `wx.onSocketOpen`：监听 WebSocket 连接打开。
- `wx.onSocketError`：监听 WebSocket 错误。
- `wx.sendSocketMessage`：发送 WebSocket 消息。
- `wx.onSocketMessage`：监听服务器返回的消息。
- `wx.closeSocket`：关闭 WebSocket 连接。
- `wx.onSocketClose`：监听 WebSocket 关闭。
- 官方建议使用 `SocketTask` 管理 WebSocket 连接，生命周期更可控。
- WebSocket 连接并发限制通常为 5 个。

示例：

```js
// 创建连接
wx.connectSocket({
  url: 'wss://example.com'
})

wx.onSocketOpen(res => {})
wx.onSocketError(res => {})

// 发送和接收消息
wx.sendSocketMessage({
  data: 'hello'
})

wx.onSocketMessage(res => {
  console.log(res.data)
})

// 关闭连接
wx.closeSocket()
wx.onSocketClose(res => {})
```

#### 4. 图片处理

- `wx.chooseImage`：从相册选择图片或使用相机拍照，成功后返回临时文件路径数组。
- `wx.previewImage`：预览图片。
- `wx.getImageInfo`：获取图片信息，如宽度、高度、路径等。
- `wx.saveImageToPhotosAlbum`：保存图片到系统相册，通常需要用户授权。
- 课程和旧题常考 `wx.chooseImage`；新项目也可能使用更通用的媒体选择能力，但本复习资料按课堂常见 API 记忆。

示例：

```js
// 选择图片
wx.chooseImage({
  count: 9,
  sizeType: ['original', 'compressed'],
  sourceType: ['album', 'camera'],
  success(res) {
    const paths = res.tempFilePaths
    console.log(paths)
  }
})

// 预览图片
wx.previewImage({
  current: url,
  urls: [url1, url2]
})

// 获取图片信息
wx.getImageInfo({
  src: url,
  success(res) {
    console.log(res.width, res.height)
  }
})

// 保存到相册
wx.saveImageToPhotosAlbum({
  filePath: path,
  success() {}
})
```

考试记忆点：

- `chooseImage` 的 `count` 常见最大值为 9。
- `sizeType` 可包含 `original`、`compressed`。
- `sourceType` 可包含 `album`、`camera`。

#### 5. 文件操作

- `wx.saveFile`：保存临时文件到本地。课堂常见口径会记“本地缓存文件上限 10MB”；当前官方文档也强调可通过 `wx.getFileSystemManager()` 进行更完整的文件系统操作。
- `wx.getFileInfo`：获取文件信息。
- `wx.getSavedFileList`：获取本地已保存文件列表。
- `wx.removeSavedFile`：删除本地已保存文件。
- `wx.openDocument`：打开文档。
- 文件操作常和 `wx.downloadFile` 配合使用：先下载得到临时路径，再保存或打开。
- 新项目还要认识 `wx.getFileSystemManager()`：获取全局唯一文件系统管理器，可进行读写、删除、目录管理等更完整的文件操作。

常见流程：

```js
wx.downloadFile({
  url: 'https://example.com/file.pdf',
  success(res) {
    wx.openDocument({
      filePath: res.tempFilePath
    })
  }
})
```

#### 6. 数据缓存

- 异步存储：`wx.setStorage`、`wx.getStorage`、`wx.removeStorage`、`wx.clearStorage`。
- 同步存储：`wx.setStorageSync`、`wx.getStorageSync`、`wx.removeStorageSync`、`wx.clearStorageSync`。
- `wx.getStorageInfo`：获取当前缓存信息，例如所有 key。
- `wx.clearStorage` / `wx.clearStorageSync`：清空所有缓存。
- `wx.removeStorage` / `wx.removeStorageSync`：删除指定 key。
- 本地缓存总空间有限，常见总上限约 10MB，单个 key 也有大小限制。
- 缓存不应当作为绝对永久数据，用户主动清理、卸载小程序或系统清理时都可能丢失。

示例：

```js
// 异步存储
wx.setStorage({
  key: 'user',
  data: userObj
})

// 同步存储
wx.setStorageSync('user', userObj)

// 异步读取
wx.getStorage({
  key: 'user',
  success(res) {
    console.log(res.data)
  }
})

// 同步读取
const user = wx.getStorageSync('user')

// 获取所有 key
wx.getStorageInfo({
  success(res) {
    console.log(res.keys)
  }
})

// 删除指定 key
wx.removeStorage({
  key: 'user'
})
wx.removeStorageSync('user')

// 清空所有缓存
wx.clearStorage()
wx.clearStorageSync()
```

考试记忆点：

- 清除所有缓存：`wx.clearStorage`。
- 删除指定缓存：`wx.removeStorage`。
- 同步 API 一般带 `Sync` 后缀。
- `Storage` 面板可以查看和管理本地缓存。

#### 7. 位置与地图

- `wx.getLocation`：获取当前位置经纬度，通常需要用户授权。
- `wx.chooseLocation`：打开地图选择位置。
- `wx.openLocation`：使用微信内置地图查看位置。

#### 8. 用户登录、授权与支付

- 获取头像可使用 `<button open-type="chooseAvatar">`。
- 获取昵称可使用 `<input type="nickname">`。
- 旧的 `wx.getUserInfo` 获取用户信息方式已不适合作为新项目主要方案。
- 登录流程：`wx.login` 获取 `code`，后端请求 `auth.code2Session` 获取 `openid` 和 `session_key`，后端再返回自定义登录态。

支付流程：

1. 小程序端发起下单请求。
2. 后端校验商品、价格、库存和用户身份。
3. 后端调用微信支付下单接口，获取 `prepay_id`。
4. 后端生成支付参数并返回给小程序端。
5. 小程序端调用 `wx.requestPayment` 拉起支付。
6. 用户完成支付。
7. 微信服务器异步通知后端支付结果。
8. 后端验签并更新订单状态。
9. 前端查询订单状态并展示结果。

支付安全点：

- 商户密钥、`appsecret`、支付私钥必须放在后端。
- 订单金额以后端数据库为准，不能相信前端传入金额。
- 支付回调必须验签。
- 订单更新要做幂等处理，防止重复通知导致重复发货。
- 请求应使用 HTTPS，并校验用户身份。

### 四、简答题

#### 1. 阐述一下滚动选择器组件的主要种类。

参考答案：

滚动选择器主要使用 `picker` 组件，通过 `mode` 属性区分类型：

- 普通选择器：`mode="selector"`，用于从一组选项中选择一项。
- 多列选择器：`mode="multiSelector"`，用于多列联动或多列选择。
- 时间选择器：`mode="time"`，用于选择时间。
- 日期选择器：`mode="date"`，用于选择日期。
- 省市区选择器：`mode="region"`，用于选择省、市、区。

#### 2. 微信中第一次打开小程序首页，依次调用了哪些生命周期函数？

参考答案：

`App.onLaunch` -> `App.onShow` -> `Page.onLoad` -> `Page.onShow` -> `Page.onReady`

说明：`onLaunch` 表示小程序初始化，`onShow` 表示小程序进入前台；页面创建时触发 `onLoad`，页面显示时触发 `onShow`，首次渲染完成后触发 `onReady`。

#### 3. 页面跳转是小程序不可或缺的功能，与此相关的路由 API 有哪些？它们之间有什么区别？

参考答案：

- `wx.navigateTo`：保留当前页面，跳转到应用内某个非 `tabBar` 页面；页面栈最多 10 层，可通过 `wx.navigateBack` 返回。
- `wx.redirectTo`：关闭当前页面，跳转到应用内某个非 `tabBar` 页面，不保留当前页面。
- `wx.switchTab`：跳转到 `tabBar` 页面，并关闭其他非 `tabBar` 页面，路径后不能带参数。
- `wx.reLaunch`：关闭所有页面，打开到应用内某个页面，可用于重启式跳转。
- `wx.navigateBack`：关闭当前页面，返回上一页或多级页面，通过 `delta` 指定返回层数。

#### 4. 小程序开发相比传统网页开发有哪些优势？

参考答案：

小程序无需用户下载安装，依托微信即可使用，入口丰富，例如搜索、扫码、分享、聊天会话等。它能调用微信生态能力，如登录、支付、定位、相机、扫码、订阅消息等，用户体验比普通网页更接近原生应用。同时，小程序提供统一的开发工具、调试环境、审核发布流程和跨 iOS/Android 的运行环境，开发和维护成本相对较低，传播也更方便。

#### 5. 请谈谈 `<block>` 标签的作用和特点。

参考答案：

`<block>` 不是一个真实组件，而是一个包装元素，常用于配合 `wx:if`、`wx:for` 包裹多个节点。它的最大特点是不会在最终页面结构中渲染成额外节点，因此适合在不破坏页面结构的情况下进行条件渲染或列表渲染。可以把它理解成 Vue 中的 `<template>`：只起模板包裹作用，不生成实际页面元素。

#### 3. 实现轮播图使用了哪些组件？每个组件的功能是什么？

参考答案：

轮播图主要使用 `swiper` 和 `swiper-item`：

#### 4. 账号注册页面需要使用哪些组件？需要设置哪些限制？

- 手机号需符合 11 位手机号格式。
- 密码需设置最小长度，可要求数字、字母组合。
- 确认密码必须与密码一致。
- 验证码不能为空且格式正确。
- 注册前需勾选用户协议或隐私协议。
- 提交前在前端校验，后端也必须再次校验，不能只依赖前端。

#### 5. 小程序项目的主要开发流程有哪些？并写出每一步的注意事项。

参考答案：

1. 注册小程序账号并获取 `AppID`。注意主体信息、类目、权限、微信支付等能力是否满足项目需求。
2. 安装微信开发者工具并创建项目。注意选择正确目录、`AppID`、基础库版本和开发模式。
3. 配置项目结构和全局文件。主要配置 `app.json` 的 `pages`、`window`、`tabBar`、网络超时等。
4. 编写页面。使用 WXML 写结构，WXSS 写样式，JS 写逻辑，JSON 写页面配置；注意数据绑定、事件绑定和组件使用规范。
5. 调试和真机预览。注意接口域名、HTTPS、权限授权、机型兼容、页面栈和性能问题。
6. 上传代码并设置体验版。注意添加体验成员，确认体验版功能完整。
7. 提交审核并发布上线。注意遵守平台规范、类目要求、隐私协议、用户授权和支付规范。
8. 上线后维护。关注错误日志、用户反馈、版本回退、数据统计和接口稳定性。

#### 6. 现在要在小程序中添加“支付”功能，需要前后端哪些交互流程？其中要注意哪些安全问题？

参考答案：

交互流程：

1. 用户在小程序端选择商品并提交订单。
2. 小程序端把商品 ID、数量、收货信息等提交给后端。
3. 后端根据数据库中的商品价格生成订单，校验库存、价格、用户身份，不能相信前端传来的金额。
4. 后端调用微信支付下单接口，获取 `prepay_id`，并生成小程序端调用支付所需的参数。
5. 后端返回 `timeStamp`、`nonceStr`、`package`、`signType`、`paySign` 等支付参数。
6. 小程序端调用 `wx.requestPayment` 发起支付。
7. 微信支付完成后通知后端，后端验签并更新订单状态。
8. 小程序端查询订单状态或根据支付结果提示用户。

安全注意事项：

- 商户密钥、`appsecret`、支付私钥等敏感信息必须放在后端，不能写在小程序端。
- 订单金额必须以后端数据库为准，不能相信前端传入金额。
- 支付回调必须验签，不能只根据前端 `success` 回调修改订单状态。
- 订单状态更新要做幂等处理，避免重复通知导致重复发货。
- 请求必须使用 HTTPS，并校验用户身份。
- 重要业务操作要防止参数篡改、重放请求和越权操作。

#### 7. 现在要在界面上添加“预约报名”功能，逻辑层和渲染层分别要做哪些修改？如何进行关联？

参考答案：

渲染层主要修改 WXML 和 WXSS：

- 在 WXML 中添加报名表单，例如姓名、手机号、预约时间、备注等输入项。
- 使用 `input`、`picker`、`button`、`form` 等组件。
- 通过 `value="{{name}}"`、`{{date}}` 等方式展示逻辑层数据。
- 通过 `bindinput`、`bindchange`、`bindsubmit` 或 `bindtap` 绑定事件。
- 在 WXSS 中设置表单布局、按钮样式、提示信息样式。

逻辑层主要修改 JS：

- 在 `data` 中定义预约表单数据，如 `name`、`phone`、`date`、`remark`。
- 编写输入事件处理函数，通过 `this.setData()` 更新页面数据。
- 编写提交函数，先校验必填项、手机号格式、预约时间等。
- 校验通过后，调用 `wx.request` 提交到服务器，或调用云函数/云数据库保存数据。
- 根据提交结果调用 `wx.showToast`、`wx.showModal` 等给用户反馈。

二者的关联方式：

WXML 通过数据绑定读取 JS 中 `data` 的值；用户触发事件后，WXML 中的 `bindxxx` 会调用 JS 中对应函数；JS 使用 `setData()` 修改数据后，渲染层自动更新。若新增独立页面，还需要在 `app.json` 的 `pages` 中注册页面路径。

## 本次核对结论

- 三张截图中的上传下载、WebSocket、图片处理、文件操作、数据缓存 API 名称整体正确，已补充到第一部分“小程序 API”。
- `wx.request`、`wx.uploadFile`、`wx.downloadFile` 官方网络能力默认超时为 60 秒，并发上限为 10 个。
- `wx.connectSocket` 创建 WebSocket 连接，官方建议使用 `SocketTask` 管理连接，WebSocket 并发限制通常为 5 个。
- `wx.chooseImage` 的 `count` 默认最大为 9，`sizeType` 包括 `original`、`compressed`，`sourceType` 包括 `album`、`camera`；当前官方也提示新场景可关注 `wx.chooseMedia`。
- 文件操作保留课堂常见 API，如 `wx.saveFile`、`wx.getSavedFileList`、`wx.removeSavedFile`、`wx.openDocument`；同时补充当前官方更强调的 `wx.getFileSystemManager()`。
- 原卷第 10 题题干末尾标注为 `D`，但按官方云函数定时触发器配置口径，正确答案应为 `B`。
- 原卷填空中控制台输出方法的大小写错误，已按 JavaScript 正确写法整理为 `console.log()`。
- 判断题中“本地缓存卸载后仍然保留”已判错；缓存不是绝对永久数据，用户清理、卸载或系统清理都可能导致丢失。
