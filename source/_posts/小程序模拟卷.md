## 模拟卷一

满分 100 分，建议用时 90 分钟。客观题 50 分，主观题 50 分。

### 一、判断题（每题 1 分，共 10 分）

1. 微信小程序具有无需下载安装、使用方便快捷的特点。
2. 微信小程序能够完全替代原生 App。
3. 开发者可以通过微信开发者工具和手机端进行预览调试。
4. 微信小程序上线通常需要经历上传、提交审核、发布等环节。
5. `block` 是一个真实渲染到页面中的组件。
6. 使用 `bindtap` 绑定点击事件时，事件默认会冒泡。

填空题：1 `Page()`；2 `App()`；3 `switchTab`；4 `code`；5 `wx.chooseAddress`。

简答题要点：

| 题号 | 参考要点                                                     |
| ---- | ------------------------------------------------------------ |
| 1    | `navigateTo` 保留当前页跳转非 tabBar；`redirectTo` 关闭当前页跳转；`switchTab` 跳转 tabBar 页面；`reLaunch` 关闭所有页面后打开某页。 |
| 2    | `form` 包裹表单控件；`input` 负责单行输入并设置 `name`；`button` 可设置提交/重置行为；通过表单提交事件把数据交给 JS 处理。 |
| 3    | Console 看日志和错误；AppData 查看页面数据；Network 查看网络请求、状态码、耗时等。 |
| 4    | `code` 是临时登录凭证，用于交给服务端换取 `openid/session_key`；前端不能把 `code` 直接当成用户身份标识或会话密钥。 |

综合题要点：

| 题号 | 参考要点                                                     |
| ---- | ------------------------------------------------------------ |
| 1    | 可用 `swiper/image` 展示轮播，`text/view` 展示价格和信息，`button` 触发加入购物车和分享，`bindtap` 处理点击；拨号用 `wx.makePhoneCall`；分享用 `onShareAppMessage`；页面加载可在 `onLoad` 根据商品 id 请求详情，用 `setData` 更新视图；必要时用 `showToast/showModal` 做反馈。 |
| 2    | `onLaunch` 不应写在 `Page()` 中，应写在 `App()` 中；`wx.login` 成功直接返回 `code` 而不是 `openid`；`wx.chooseAddress` 不必嵌在 `getLocation` 成功后才能调用，且两者授权不同；跳转 tabBar 页面应用 `wx.switchTab` 而不是 `navigateTo`；`upload` 前没有先 `chooseImage`，`imgs[0]` 可能为空；`uploadFile` 缺少 success/fail/complete 反馈；涉及位置权限应在 `app.json` 配置 `scope.userLocation` 描述；上传前应校验文件路径和网络状态。 |
